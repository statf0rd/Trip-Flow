# Агентская шпаргалка — TripFlow (Triloo)

Коротко: интерактивный помощник в поездках с совместным планированием маршрута, расходами, геошарингом и офлайн‑синхронизацией через QR.

## Что уже работает
- Навигация: Trip list + создание личных/групповых поездок, TripDetails с табами обзор/план/карта(заглушка)/расходы; дни создаются автоматически по диапазону дат.
- Места: добавление/редактирование с автодополнением через mock `PlacesService`, таймлайн по дням, отметка посещения, reorder, просмотр деталей (`ui/tripdetails`, Room).
- Расходы: добавление и список в табе расходов, категории/валюта, сумма в базовой валюте; репозиторий считает summary и долги, хранит курсы валют (Room). В UI сплит пока PAYER_ONLY.
- Групповые функции: инвайты и QR (`ui/invite`, `ui/qr`), экран групповых поездок с входом по коду/скану (`ui/grouptrips`), участники и их координаты хранятся в `TripRepository`.
- QR/Relay: Triloo Relay для офлайн‑синхронизации через QR чанки, лог удалений (`data/relay`, `ui/relay`); общий сканер QR на ZXing (`ui/qr`).
- Настройки/профиль/авторизация: экран настроек с темой и переходом к политике (`ui/settings`), тема в DataStore (`data/settings`); политика из `res/raw/privacy_policy`; профиль и anon device id в DataStore (`data/user`); фейковый AuthFlow + `FakeAuthRepository` (`ui/auth`, `data/auth`).
- Сервисы/утилиты: маршрутный оптимизатор NearestNeighbor + рекомендации (`data/route`), мок‑Places API (`data/places`), калькулятор heatmap по отзывам категории (`data/heatmap`), общие компоненты/форматтеры (`ui/components`, `TripFormatters.kt`).

## Ближайшие цели
- Реальная интеграция Google Maps/Places (карта в TripDetails, координаты/фото, nearby).
- Directions/ETA в маршрутном оптимизаторе и UI, авто‑маршруты по дням.
- Курсы валют из сети + полноценные сплиты/балансы в UI.
- Онлайн‑синхронизация (backend/Firebase), живой геошаринг, конфликт‑резолюция.
- Auth c реальным провайдером, уведомления, OCR чеков, персональные рекомендации.

## Архитектура и папки
- UI: `ui/trips`, `ui/tripdetails`, `ui/grouptrips`, `ui/invite`, `ui/relay`, `ui/qr`, `ui/auth`, `ui/settings`, `ui/components`, `ui/theme`, навигация в `ui/navigation`.
- Data: Room (`data/local`), репозитории (`data/repository`), mock сервисы `data/places` и `data/route`, auth stub `data/auth`, профиль `data/user`, настройки `data/settings`.
- DI: Hilt модули в `di`. Точка входа `TrilooApp.kt`, `MainActivity.kt`.

## Ключевые модели и сценарии
- Поездка: `Trip`, дни `TripDay`, точки `Place`, расходы `Expense`, участник `Participant` (в `data/model`).
- Детали поездки: табы обзор/план/карта/расходы управляются через `TripDetailsTabs.kt` и соседние экраны.
- QR: инвайты и Relay используют ZXing для генерации/сканирования пакетов.

## Интеграции и ключи
- Google Maps/Places: ключ в `local.properties` (`MAPS_API_KEY=...`), карты пока заглушка.
- Offline/online синхронизация: сейчас только QR Relay; сетевой backend в планах.

## Дизайн-система
- Цвета: Coral `#FF6B5B`, Teal `#2DD4BF`, Golden `#FBBF24`, нейтральные Slate `#0F172A`–`#F8FAFC`.
- Шрифты: Outfit для заголовков, DM Sans для текста.
- Тема и токены лежат в `app/src/main/java/com/triloo/ui/theme`.

## Шпаргалка по сборке и проверкам
- Android Studio Hedgehog 2024.1+, JDK 17, minSdk 26.
- Сборка: `./gradlew assembleDebug`
- Тесты/проверки: `./gradlew testDebugUnitTest lint`
- Перед картой убедиться, что API ключ прописан и включены Maps/Places в консоли.
